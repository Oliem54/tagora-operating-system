# TBF — TAGORA Business Framework

## Objectif

Définir le cadre officiel des règles métier communes transverses à l'écosystème TAGORA.

## Portée

Domaines métier partagés — employés, compagnies, permissions, clients, fournisseurs, commissions, pay plans, documents, inventaire, congés, horodateur, notifications, multi-compagnies.

## Principes

- **Règles communes documentées** — Une seule définition des concepts transverses.
- **Métier dans les applications** — TBF définit ; les applications implémentent leur spécificité.
- **Cohérence inter-modules** — Même sémantique métier partout.
- **Multi-compagnies natif** — Conçu pour le contexte TAGORA.
- **Évolution contrôlée** — Changements métier transverses via gouvernance.

## Règles

- Documenter toute règle métier commune avant implémentation dans un module.
- Ne pas dupliquer une règle TBF dans un module sans justification.
- Distinguer règles communes (TBF) et règles spécifiques au module.
- Valider les impacts inter-modules avant modification d'une règle TBF.
