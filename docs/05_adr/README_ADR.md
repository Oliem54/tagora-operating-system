# README — Architecture Decision Records (ADR)

## Objectif

Expliquer le rôle, l'usage et la gouvernance des ADR dans le TAGORA Knowledge System (TKS).

## Portée

Toutes les décisions architecturales et méthodologiques significatives de l'écosystème TAGORA.

## Principes

- TKS est la mémoire permanente officielle.
- Chaque décision significative est traçable, datée et justifiée.
- Les ADR sont immuables une fois acceptés — les changements produisent un nouvel ADR ou un amendement documenté.
- Format uniforme via `ADR_TEMPLATE.md`.

## Règles

- Créer un ADR avant ou immédiatement après une décision architecturale significative.
- Numéroter les ADR de façon séquentielle (ex. `ADR-0001-titre.md`).
- Stocker les ADR dans `docs/05_adr/`.
- Référencer les ADR depuis les documents standards concernés.
