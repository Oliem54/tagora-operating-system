# Standard — QA

## Objectif

Définir le standard officiel de qualité et de tests applicable à tous les modules TAGORA.

## Portée

Processus QA, tests manuels, automatisés, comptes QA, données QA et validation pré-production.

## Principes

- QA avant production — non négociable.
- Scénarios reproductibles par toute l'équipe.
- Playwright comme standard d'automatisation UI.
- Comptes et données QA dédiés et maintenus.
- Smoke tests obligatoires avant release.

## Règles

- Maintenir des comptes QA conformes au standard TQF.
- Documenter et versionner les scénarios QA.
- Exécuter smoke tests dans le pipeline CI (TRF).
- Référencer `docs/06_qa/` pour les spécifications détaillées.
