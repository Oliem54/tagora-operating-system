# Standard — Documentation

## Objectif

Définir le standard officiel de documentation applicable à l'écosystème TAGORA — technique, métier, utilisateur et méthodologique.

## Portée

Toute documentation produite par les projets TAGORA, en complément du TKS et du TOS.

## Principes

- **Documentation avant développement** — Première étape de l'ordre méthodologique officiel.
- Clarté, structure et maintenabilité.
- Une source de vérité par sujet (TKS pour les décisions).
- Templates uniformes (`docs/99_templates/`).
- Documentation vivante — mise à jour avec le code.

## Règles

- Utiliser les templates TOS pour nouveaux standards et décisions.
- Documenter avant toute implémentation significative.
- Référencer les ADR pour les décisions architecturales.
- Maintenir README et changelog à jour par module.

## Éléments à documenter plus tard

- Types de documentation requis par module
- Niveaux de détail (overview, guide, référence)
- Processus de revue documentaire
- Outils de publication (à décider)
- Politique de langue (FR/EN)
