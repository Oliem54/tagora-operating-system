# Template — Release notes

## Objectif

Modèle standard pour rédiger des **release notes** TAGORA.

## Portée

Communication release — tous modules. **Ne publie rien automatiquement.**

Standard : [RELEASE_NOTES_STANDARD.md](../11_release/RELEASE_NOTES_STANDARD.md)

---

## Modèle complet

Copier et remplir :

```markdown
# Release notes — [Module] v[X.Y.Z]

| Champ | Valeur |
|---|---|
| **Titre** | [Titre court lisible] |
| **Version** | v[X.Y.Z] |
| **Date** | [YYYY-MM-DD] |
| **Module** | [TAGORA Time / ...] |

## Résumé

[2–3 phrases — valeur livrée et contexte]

## Nouveautés

- [Point 1 — langage accessible]
- [Point 2]

## Corrections

- [Correction 1]
- [Correction 2]

## Changements importants

- [Comportement modifié nécessitant attention]

## Impact utilisateur

[Qui est concerné — employés, superviseurs, direction — et comment]

## Actions requises

- [ ] [Action 1 — ex. former superviseurs]
- [ ] [Action 2 — ou « Aucune action requise »]

## Problèmes connus

- [Limitation connue — ou « Aucun problème connu majeur »]

## Support

[Procédure contact support — sans secret]

## Notes internes

[Réservé équipe — détails techniques, rollback, monitoring]
```
