# Design Premium 2027

## Objectif

Définir la direction **design premium TAGORA** à horizon 2027, guidant le TAGORA Design System (TDS) pour une identité cohérente, moderne et durable.

## Portée

Vision design, principes visuels et expérience premium — toutes applications TAGORA, en commençant par **TAGORA Time** (pilote).

---

## Définition du design premium TAGORA

Le **design premium TAGORA** est une expérience visuelle et interactionnelle qui transmet :

- **Confiance** — L'utilisateur sent que l'outil est fiable et professionnel.
- **Clarté** — L'information importante est immédiate ; rien n'est caché par le décor.
- **Efficacité** — L'utilisateur accomplit sa tâche sans friction inutile.
- **Cohérence** — Chaque module TAGORA semble appartenir au même univers.

Ce n'est pas un excès d'effets visuels. C'est une **qualité perçue** de logiciel professionnel moderne — soigné, lisible, prévisible — **sans copier une marque existante**.

---

## Principes directeurs

TAGORA doit toujours être :

| Principe | Signification |
|---|---|
| **Simple** | Peu d'éléments à la fois ; pas de surcharge cognitive. |
| **Rapide** | Perception de réactivité ; parcours courts pour les actions fréquentes. |
| **Intuitif** | Compréhensible sans manuel ; libellés clairs. |
| **Professionnel** | Ton sérieux, adapté au travail et à la gestion. |
| **Épuré** | Espace, hiérarchie, pas de bruit visuel. |
| **Moderne** | Contemporain sans être éphémère ; durable sur plusieurs années. |

Compléments constitutionnels : **Accessible**, **Lisible**, **Premium**.

---

## Règles fondamentales

### Aucune interface surchargée

- Limiter le nombre d'actions visibles simultanément.
- Regrouper les fonctions secondaires (menus, vues détaillées).
- Éviter les tableaux de bord « fourre-tout » sans hiérarchie.

### Informations critiques visibles immédiatement

- Statut actuel (ex. punch ouvert, en attente de validation).
- Action principale de l'écran (ex. punch in, valider).
- Alertes et erreurs blocking — jamais uniquement dans un coin discret.

### Détails complexes en vues secondaires

- Historique détaillé, paramètres avancés, rapports analytiques → écrans ou panneaux secondaires.
- L'écran principal reste **actionnable en quelques secondes**.

---

## Inspiration générale

L'ambition TAGORA s'inspire du **niveau de qualité perçue** des grands écosystèmes professionnels (suite bureautique unifiée, applications métier premium) :

- Cohérence entre modules.
- Reconnaissance immédiate de l'univers produit.
- Simplicité d'usage au quotidien.

**TAGORA ne copie pas** Google, Microsoft ou toute autre marque. Il définit **sa propre signature** — documentée dans le TDS.

## Application progressive à TAGORA Time

| Priorité design Time | Objectif |
|---|---|
| Navigation claire | Employé, superviseur, direction, admin — chacun sait où il est. |
| Statuts visuels | Punch actif, terminé, retard, validation — compréhensibles sans légende longue. |
| Dashboard employé | Action principale (pointage) évidente. |
| Dashboard direction | Synthèse lisible ; détails en drill-down. |
| Erreurs | Messages compréhensibles, visibles. |
| Desktop / mobile | Cohérence de principes ; adaptation responsive. |

Le design Time est **documenté dans TOS** avant toute refonte UI importante dans le dépôt Time.
